# 会员积分系统 — 详细需求

> **版本**: V2.1.0  
> **适用阶段**: 积分系统开发  
> **依赖**: [05-数据库设计.md](05-数据库设计.md)（会员模块表结构）  
> **Token 预算**: 中等（约 400 行）

---

## 模块概述

会员积分系统面向微信小程序用户端和 Web 管理后台，提供会员注册、积分获取/使用、商品兑换、积分规则配置等功能。

**核心目标**：通过积分体系增加用户粘性，形成"消费-积分-复购"闭环。

## 数据表

> 完整字段定义见 [05-数据库设计.md](05-数据库设计.md) 的"会员模块"章节。

|------|------|----------|
| `tb_member` | 会员表 | member_id, name, phone, level, points |

| `tb_admin` | 管理员表 | username, password, role |
| `tb_points_rule` | 积分规则表 | earn_rate, deduct_rate, valid_months |

## 功能需求

### 3.1 微信快捷登录/会员注册

**用户故事**
```
作为 店铺顾客，我希望 通过微信一键登录，以便 无需注册账号密码即可使用会员功能。
作为 新会员，我希望 完善个人信息后获得赠送积分，以便 激励我完善资料。
```

**界面及交互**

| 元素 | 类型 | 必填 | 校验规则 |
|------|------|------|---------|
| 微信授权按钮 | 按钮 | — | 调起微信授权 |
| 姓名 | 文本输入 | 否 | 1-20 字符 |
| 手机号 | 文本输入 | 是 | 11 位手机号格式 |
| 运动偏好 | 单选 | 否 | 全部/羽毛球/网球 |

**业务流程**

```mermaid
flowchart TD
    A[用户点击微信登录] --> B[调起微信授权]
    B -->{授权成功?}
    B -->|否| C[提示授权失败]
    B -->|是| D{是否新会员?}
    D -->|是| E[创建会员记录]
    E --> F[跳转完善信息页]
    D -->|否| G[更新登录时间]
    G --> H[跳转首页]
    F --> I[用户填写信息]
    I --> J[保存+赠送10积分]
    J --> H
```

**API 接口**

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/api/auth/wx-login` | 微信登录 |
| POST | `/api/auth/register` | 完善信息注册 |
| PUT | `/api/member/profile` | 更新个人资料 |
| GET | `/api/member/profile` | 获取个人资料 |

### 3.2 积分管理

**用户故事**
```
作为 店铺管理员，我希望 手动调整会员积分（补赠/扣除），以便 处理特殊情况。
作为 店铺管理员，我希望 查看积分明细，以便 核对积分变动情况。
作为 会员，我希望 使用积分抵扣消费，以便 享受会员优惠。
```

**界面及交互**

| 元素 | 类型 | 必填 | 校验规则 |
|------|------|------|---------|
| 会员搜索 | 文本输入 | 是 | 手机号或会员ID |
| 操作类型 | 下拉选择 | 是 | 补赠/扣除 |
| 积分数量 | 数字输入 | 是 | 1-99999 |
| 调整原因 | 文本域 | 是 | 1-200 字符 |

**API 接口**

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/points/records` | 积分明细（分页） |
| POST | `/api/points/adjust` | 积分调整（补赠/扣除） |
| POST | `/api/points/redeem` | 积分核销（抵扣消费） |
| GET | `/api/points/balance` | 积分余额 |

**数据字典**

| type | 说明 |
|------|------|
| earn | 获得积分 |
| deduct | 扣除积分 |

### 3.3 积分兑换管理

**用户故事**
```
作为 会员，我希望 用积分兑换商品或服务，以便 享受会员权益。
作为 店铺管理员，我希望 管理可兑换的商品/服务，以便 灵活配置兑换内容。
```

**界面及交互**

| 元素 | 类型 | 必填 | 校验规则 |
|------|------|------|---------|
| 商品名称 | 文本输入 | 是 | 1-100 字符 |
| 商品描述 | 文本域 | 是 | 1-500 字符 |
| 所需积分 | 数字输入 | 是 | ≥ 1 |
| 库存 | 数字输入 | 是 | ≥ 0 |
| 商品类型 | 下拉选择 | 是 | product/service |
| 状态 | 下拉选择 | 是 | active/inactive |

**API 接口**

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/exchange/items` | 兑换商品列表 |
| POST | `/api/exchange/item` | 添加兑换商品 |
| PUT | `/api/exchange/item/{id}` | 更新兑换商品 |
| DELETE | `/api/exchange/item/{id}` | 删除兑换商品 |
| POST | `/api/exchange/redeem` | 积分兑换 |
| GET | `/api/exchange/records` | 兑换记录 |

**数据字典**

| status | 说明 |
|--------|------|
| pending | 待核销 |
| redeemed | 已核销 |
| cancelled | 已取消 |

### 3.4 积分规则设置

**API 接口**

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/points/rule` | 获取积分规则 |
| PUT | `/api/points/rule` | 更新积分规则 |

**规则字段**

| 字段 | 说明 | 示例 |
|------|------|------|
| earn_rate | 获取比例（1元=N分）| 1 |
| deduct_rate | 抵扣比例（N积分=1元）| 10 |
| valid_months | 有效期（月）| 12 |

**规则修改说明**：
- 积分规则修改仅影响 **新获得** 的积分，已有积分不受影响
- 修改后，新消费按新规则计算积分，已获得的积分余额保持不变
- 积分有效期按"获得时间"计算，不按"规则修改时间"重新计算
- 规则修改需记录操作日志（`tb_operation_log`），包括修改人、修改时间、修改前后值

## 验收标准

| 功能 | 验收条件 | 优先级 |
|------|---------|--------|
| 微信登录 | 授权成功后自动创建会员并跳转首页 | P0 |
| 积分查询 | 显示当前余额和最近 10 条明细 | P0 |
| 积分补赠 | 管理员补赠后积分实时到账，会员收到通知 | P0 |
| 积分兑换 | 积分足够时兑换成功，生成兑换码 | P0 |
| 积分不足 | 积分不足时提示"积分不足"，不允许兑换 | P0 |
| 手机号重复 | 注册时手机号已存在，提示"该手机号已注册" | P0 |
