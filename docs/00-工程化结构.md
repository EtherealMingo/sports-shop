### 2.2 分层规范

| 层 | 包名 | 职责 | 命名规范 |
|----|------|------|---------|
| Controller | `controller` | 接收请求、参数校验、调用 Service | `XxxController` |
| Service | `service` | 业务逻辑、事务管理 | `XxxService` / `XxxServiceImpl` |
| Mapper | `mapper` | 数据库操作 | `XxxMapper` / `XxxMapper.xml` |
| Entity | `entity` | 数据库表映射 | `Xxx.java` |
| DTO | `dto` | 数据传输对象 | `XxxDTO.java` |
| VO | `vo` | 视图对象 | `XxxVO.java` |

### 2.3 接口规范

| 规范 | 说明 | 示例 |
|------|------|------|
| URL 前缀 | `/api/{模块}` | `/api/member`, `/api/wire` |
| 请求方式 | RESTful | GET/POST/PUT/DELETE |
| 返回格式 | `Result<T>` | `{ code: 200, message: "成功", data: T }` |
| 分页参数 | `current`, `pageSize` | `?current=1&pageSize=10` |
| 分页返回 | `PageResult<T>` | `{ data: [], total: 100, current: 1, pageSize: 10 }` |

## 5. 统一规范

### 5.1 API 接口命名

| 操作 | HTTP 方法 | URL 模式 | 说明 |
|------|----------|---------|------|
| 列表 | GET | `/api/{module}/list` | 分页列表 |
| 详情 | GET | `/api/{module}/{id}` | 单条详情 |
| 新增 | POST | `/api/{module}` | 创建记录 |
| 更新 | PUT | `/api/{module}/{id}` | 更新记录 |
| 删除 | DELETE | `/api/{module}/{id}` | 软删除 |
| 批量 | POST | `/api/{module}/batch` | 批量操作 |

### 5.2 统一返回格式

```typescript
interface Result<T> {
  code: number;        // 200=成功, 500=失败
  message: string;     // 提示信息
  data: T;             // 数据
}

interface PageResult<T> {
  data: T[];           // 列表数据
  total: number;       // 总条数
  current: number;     // 当前页
  pageSize: number;    // 每页条数
}
```

### 5.3 错误码定义

| 错误码 | 说明 | HTTP 状态码 |
|--------|------|------------|
| 200 | 成功 | 200 |
| 400 | 参数错误 | 400 |
| 401 | 未认证 | 401 |
| 403 | 无权限 | 403 |
| 404 | 资源不存在 | 404 |
| 500 | 服务器错误 | 500 |
| 1001 | 积分不足 | 400 |
| 1002 | 库存不足 | 400 |
| 1003 | 预约冲突 | 400 |
| 1004 | 二维码无效 | 400 |
| 1005 | 二维码已使用 | 400 |

### 5.4 命名规范

| 类型 | 规范 | 示例 |
|------|------|------|
| 数据库表 | `tb_` 前缀 + 小写单数 | `tb_member`, `tb_wire` |
| 字段 | snake_case | `create_time`, `member_id` |
| Java 类 | PascalCase | `MemberController` |
| Java 方法 | camelCase | `getMemberList()` |
| React 组件 | PascalCase | `MemberModal.tsx` |
| Vue 组件 | PascalCase | `PointsCard.vue` |
| 路由 | kebab-case | `/member-list` |
| 变量 | camelCase | `memberList` |
| 常量 | UPPER_SNAKE | `MAX_RETRY_COUNT` |

### 5.5 请求封装

**前端请求拦截器**：
- 自动添加 JWT Token
- 自动处理 401 跳转登录
- 统一错误提示
- Loading 状态管理

**后端响应拦截器**：
- 统一异常处理
- 自动填充审计字段
- 操作日志记录

## 6. 开发优先级

### Phase 1: 基础骨架（1-2 周）
1. 后端项目初始化（Spring Boot + MyBatis Plus）
2. 数据库建表（init.sql）
3. 小程序项目初始化（uniapp）
4. 管理后台项目初始化（Ant Design Pro）
5. 基础组件开发（表格、表单、弹窗）

### Phase 2: 会员积分系统（2-3 周）
1. 微信登录 + JWT 认证
2. 会员管理 CRUD
3. 积分管理（查询/调整/核销）
4. 积分兑换（商品/记录）
5. 积分规则设置

### Phase 3: 线材管理系统（2-3 周）
1. 线材 CRUD + 供应商管理
2. 进货管理（批次计价）
3. 消耗/报废记录
4. 库存查询 + 低库存预警

### Phase 4: 二维码 + 预约（2 周）
1. 二维码生成 + 打印 + 扫码
2. 服务预约 + 进度追踪
3. 服务历史 + 个性化推荐

### Phase 5: 财务增强（1-2 周）
1. 财务报表
2. 利润分析 + 技师绩效
3. 应付账款 + 价格趋势

### Phase 6: 系统打通 + 优化（1 周）
1. 积分-线材系统打通
2. 灰度发布
3. 性能优化 + 体验优化
