# 体育器材服务店铺管理系统 — 项目概述

> **版本**: V2.1.0  
> **适用阶段**: 项目启动、全局概览  
> **Token 预算**: 轻量（约 200 行）

---

## 产品概述

面向体育器材服务店铺的综合管理系统，核心目标：
- 增加用户粘性、回馈老客户（会员积分）
- 精细化运营、数据驱动决策（线材管理 + 财务分析）

**业务核心**：缠线服务（高频刚需）+ 器材售卖（低频高客单）+ 积分体系闭环 + 线材料级追踪

## 技术栈

| 分层 | 技术 | Skill |
|------|------|-------|
| 小程序前端 | uniapp + Vue 3 + UnoCSS + TnUI | `ui-ux-pro-max` + `frontend-design` |
| 管理后台前端 | React 19 + Ant Design Pro | `ui-ux-pro-max` + `frontend-design` |
| 后端 | Spring Boot 3.2 + MyBatis Plus + MySQL 8.0 | `db-migrator` + `api-tester` |
| 认证 | JWT | - |

## 灰度发布策略

| 阶段 | 范围 | 时间 | 回滚方案 |
|------|------|------|---------|
| Phase 1 | 内部员工试用 | 上线后 1 周 | 数据库回滚 + 代码回滚 |
| Phase 2 | 20% 会员开放 | 上线后 2 周 | 功能开关关闭 + 数据库回滚 |
| Phase 3 | 50% 会员开放 | 上线后 3 周 | 功能开关关闭 + 数据库回滚 |
| Phase 4 | 全量开放 | 上线后 4 周 | — |

**回滚触发条件**：
- 核心功能出现 P0 Bug（如积分计算错误、库存扣减异常）
- 系统响应时间超过 3 秒
- 用户投诉率超过 5%
