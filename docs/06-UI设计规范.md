## 设计流程

```
1. 分析需求 → 提取产品类型/行业/风格关键词
2. 生成设计系统 → ui-ux-pro-max --design-system
3. 补充查询 → 按 domain 查询细节（style/typography/color/ux/chart）
4. 技术栈指南 → --stack react/vue/html-tailwind
5. 实现代码 → 遵循 frontend-design 美学指南
6. 交付前检查 → 视觉质量 + 交互 + 对比度 + 响应式
```

## 小程序端设计规范

### 主题色彩

```css
/* 主色调 - 运动活力橙 */
--primary: #FF6B35;
--primary-light: #FF8C61;
--primary-dark: #E55A2B;

/* 辅助色 */
--secondary: #00D9FF;
--accent: #FFD700;
--success: #00E676;
--warning: #FFC107;
--error: #FF5252;

/* 背景色 */
--bg-primary: #0A0E27;
--bg-secondary: #12183A;
--bg-tertiary: #1A237E;

/* 文字色 */
--text-primary: #FFFFFF;
--text-secondary: #B0BEC5;
--text-muted: #78909C;
```

### 视觉风格

- **整体风格**：深色模式 + 高对比度 + 运动科技感
- **渐变运用**：橙-蓝渐变体现活力
- **圆角设计**：大圆角（16-24rpx）
- **动效**：快速响应（200-300ms）、弹性缓动
- **图标风格**：线性图标 + 品牌色点缀

### 字体规范

- 标题：PingFang SC Bold，字号 36-48rpx
- 正文：PingFang SC Regular，字号 28-32rpx
- 数字：DIN Alternate Bold / Oswald

## 通用设计原则（frontend-design）

### 字体

- **禁止**：Arial / Inter / Roboto / system fonts
- **推荐**：Barlow Condensed / Playfair Display / DM Serif Display / Space Grotesk
- **搭配**：标题用展示型字体 + 正文用精致型字体

### 色彩

- 主色 + 强调色，避免均匀分布
- 使用 CSS 变量保持一致性
- 深色/浅色模式均需测试

### 动效

- CSS 优先，React 用 Motion 库
- 150-300ms 微交互
- 重点：页面加载时的交错动画（animation-delay）

### 布局

- 不对称 / 重叠 / 对角线 / 打破网格
- 大量留白 OR 受控密度（二选一）

### 背景

- 渐变网格 / 噪点纹理 / 几何图案 / 分层透明
- 避免纯色背景

### 交互

- 所有可点击元素必须有 `cursor-pointer`
- Hover 用颜色/透明度过渡，禁止 scale
- 玻璃卡片 light 模式用 `bg-white/80`

### 无障碍

- 文字对比度 ≥ 4.5:1
- 图片有 alt 文字
- 表单有 label
- 尊重 `prefers-reduced-motion`
- 响应式：375px / 768px / 1024px / 1440px

## 交付前检查清单

### 视觉质量
- [ ] 无 emoji 图标（使用 SVG：Heroicons/Lucide）
- [ ] 所有图标来自统一图标集
- [ ] Hover 状态不导致布局偏移
- [ ] 使用主题颜色而非 var() 包装

### 交互
- [ ] 所有可点击元素有 `cursor-pointer`
- [ ] Hover 状态有清晰视觉反馈
- [ ] 过渡平滑（150-300ms）
- [ ] 焦点状态可见

### 对比度
- [ ] 文字对比度 ≥ 4.5:1
- [ ] 玻璃卡片 light 模式 `bg-white/80` 以上
- [ ] 边框在两种模式下均可见

### 响应式
- [ ] 375px / 768px / 1024px / 1440px
- [ ] 移动端无水平滚动
- [ ] 固定导航栏不遮挡内容
